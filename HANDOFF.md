## Context Notes

- **`GeneralDetailReportQueryRq` is the new load-bearing wire request for Phase 11 detail reports.** Schema order pinned in [tests/builder-emit-order.test.ts](tests/builder-emit-order.test.ts) — if a future variant introduces a new filter (e.g. `ReportClassFilter`), slot it carefully. Current order is inferred from QBXML 16.0 patterns; not yet line-by-line verified against the actual XSD. The schema-order test catches reorderings on the same pinned children.
- **Detail-half live adapter is reused from `CustomDetailReportQueryRq`'s adapter.** Same `adaptLiveCustomDetailReportRet` — wire shape (ColDesc + ReportData row tree) is structurally identical. This is the cheap reuse that made the paired delivery viable.
- **Summary-half live adapter is the new "flat" fork in `adaptLiveReportRet`.** Triggers ONLY when neither `PNL_SECTION_NAMES` nor `BS_SECTION_NAMES` matches any TextRow AND there are actual data rows. Empty data short-circuits to `Sections: []`. The detection is locale-stable (no ReportTitle dependency) but the test file's regression pin against P&L+Income TextRows guarantees the existing PnL path is unaffected.
- **Sim aggregation matches QB's actual report math.** `line.Amount` (not `txn.Subtotal` and not `txn.TotalAmount`) so sales-tax-only lines and zero-amount lines drop naturally. CreditMemo lines subtract on both summary (per-customer total) and detail (per-row Amount). Pinned in tests.
- **`buildReportRequest` is backwards-compatible.** P&L and BS callers that don't pass `entityFilter` get the original wire shape — pinned by a `expect(plain).not.toContain("ReportEntityFilter")` assertion in the schema-order test.
- **Item line resolution in the detail walker falls through to "Uncategorized Income"** when the sim's seed Service items don't carry an `IncomeAccountRef`. This is the documented fallback (matches `qb_pnl_report`'s behavior). In live mode where items DO carry IncomeAccountRef, the real GL account name surfaces — same line resolution code path as `qb_pnl_report` (`resolveLineAccount` in [src/session/simulation-store.ts](src/session/simulation-store.ts)).
